# Task 11: 飞书集成

## Feature
将飞书 CLI 操作封装为 MCP Tool，集成到 Agent Runtime，支持多维表格、文档、日历、消息等功能。

## 优先级
P2 - 产品差异化功能

## 集成架构

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  飞书 MCP     │     │  MCP Client  │     │  Agent       │
│  Server       │────▶│  (Rust)      │────▶│  Runtime     │
│  (Sidecar)    │◀────│              │◀────│              │
└──────┬───────┘     └──────────────┘     └──────────────┘
       │
       ▼
┌──────────────┐
│  飞书开放平台  │
│  多维表格     │
│  文档        │
│  日历        │
│  消息/机器人  │
│  审批流      │
└──────────────┘
```

## 验收标准
- [ ] 飞书 MCP Server 封装: CLI → MCP 协议适配
- [ ] 多维表格: CRUD 操作、数据同步到本地
- [ ] 文档: 创建/编辑/读取，Markdown 双向同步
- [ ] 日历: 事件创建/查询，任务排期
- [ ] 消息机器人: Agent 进度通知推送到飞书
- [ ] 认证: 飞书 App 凭证管理
- [ ] 配置 UI: 飞书集成配置界面
- [ ] Ideas 同步: 飞书多维表格 ↔ 本地 Ideas
- [ ] 任务同步: 飞书多维表格 ↔ 本地 Tasks

## 实现路径
1. **Phase 1**: 飞书 CLI 封装为 MCP Server (Python/Node sidecar)
2. **Phase 2**: 注册到 MCP Client，Agent 可调用飞书操作
3. **Phase 3**: 双向数据同步 (多维表格 ↔ SQLite)
4. **Phase 4**: 飞书机器人作为通知通道
